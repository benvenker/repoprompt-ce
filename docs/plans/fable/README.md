# Implementation Plans — headless Linux extraction of the RepoPrompt CE context engine

Generated by the improve skill on 2026-06-11 at commit `1db9bbc`. Goal:
run RepoPrompt CE's context-engineering backend (deterministic tools +
`oracle_send` + `context_builder`) on a Linux VPS as a standalone MCP
server/CLI, with any-model support via OpenRouter and pluggable discovery
agents. This is a **private fork**: upstream mergeability is explicitly
waived; the macOS app build is kept green only as a regression net.

Execute in the order below unless dependencies say otherwise. Each executor:
read the plan fully before starting, honor its STOP conditions, and update
your row when done. Background evidence (file:line) lives in
`docs/investigations/context-harness-extraction-feasibility-2026-06-11.md`.

  engine must be a library before any server can exist.
- 001 → 004: 004 inherits 001's validated Docker image and pass/fail findings
  for SwiftTreeSitter/CSwiftPCRE2; if 001 is BLOCKED, 004 is blocked.

  target as each other (`Sources/RepoPromptHeadlessServer`) — serialize 005
  before 006.

  are mutually independent in content (007: main/server/bridge; 008: session

  010: schemas + tests), but they all build the same target — serialize
  executors within `Sources/RepoPromptHeadlessServer` (one at a time),

## Findings considered and rejected

- **Port the full app provider stack (16 providers, SwiftOpenAI/SwiftAnthropic)**
  — rejected: macOS-URLSession-based and view-model-hosted; one
  OpenAI-compatible SSE client over AsyncHTTPClient covers every model via
  OpenRouter/custom base URLs (plan 005).
- **Port `MCPWindowToolDependencies` + app tool providers wholesale** —
  rejected: the bundle stores `PromptViewModel`/`WorkspaceManagerViewModel`
  (verified at `MCPWindowToolDependencies.swift:224-227`); fresh narrow
  handlers over ContextCore services are smaller than adapting ~130 closures.
- **Rewrite the engine in TypeScript/Rust as a Pi extension** — rejected:
  ~60 KLOC incl. tree-sitter integration; Pi consumes MCP via the
  `pi-mcp-adapter` extension instead (plan 006 documents the hookup).
- **Native Swift agent loop for discovery (tool-calling against OpenRouter)**
  — deferred, not planned: external agent CLIs (Claude Code/Codex/OpenCode/
  Pi) already provide the loop; revisit only if CLI orchestration proves
  unreliable.
- **inotify file watcher on Linux** — deferred: rescan-on-demand with
  debounce is sufficient for agent workloads (plan 004 leaves a marked seam).
- **`git` MCP tool, `apply_edits`, slices, multi-workspace/tab semantics,
  approvals UX** — deferred from v1 scope; tracked in plan maintenance notes.
- **Upstream-mergeable refactor posture** — explicitly waived by the
  repository owner on 2026-06-11 (private fork, move fast).
- **App-parity ops for headless `agent_run`/`agent_manage`** (`steer`,
  `respond`, `session_ids` batch poll/wait, `extract_handoff`) — deferred,
  not planned (2026-06-12 review): real utility but below the 007–012 fixes;
  `session_ids` batch wait and `extract_handoff` are the first candidates if
  VPS fan-out patterns demand them.
- **Auto-expiry/retention sweep for terminal agent sessions** — deferred
  from plan 008 (client-driven `cleanup_sessions` is the v1 contract).
