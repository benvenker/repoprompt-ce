## Goal

Merge multiple reviewer reports into one conservative, actionable polishing recommendation for this bead.

Use the project-local workflow policy before synthesizing: read
`.smithers/skills/beads-polish-v3.md` and
`.smithers/skills/beads-from-plan-v1.md`, then apply their split/merge
discipline, quality-gate rules, and polish-loop guidance. Broader Better Beads
material may inform taste, but local workflow policy wins. Do not treat long
beads as bad merely because they are long; run the split test first.

## Classification discipline

Classify every useful finding as exactly one of:

- Same-contract detail: improves the same outcome's success criteria, failure behavior, validation, anchors, non-goals, or closure evidence.
- New independent behavior: deserves a split/new bead only if it has its own observable outcome and reviewable implementation PR/commit.
- Graph correction: dependency edge, parent closure contract, label/frontier, status, or unsafe parallelism fix.
- Readability-only: wording or BV formatting that does not change behavior.
- No-op: reviewer preference that should not mutate the graph.

For long-child warnings, run the split test before prose compaction:

- Does the long section describe a separate observable system truth?
- Can that behavior land, be reviewed, and be verified independently?
- Would splitting it change dependency edges, parent order, or the true ready frontier?

For dense single beads, preserve an auditable split decision:

- Identify the behavior atoms the reviewers found or implied.
- If all atoms prove one small sub-plan outcome, recommend adding or retaining a
  compact "Split decision" note in the bead so future polish/judge passes can
  see why no parent/child graph exists.
- If two or more atoms can land as independent behavior/system truths, classify
  the result as a graph correction and recommend a parent closure bead plus
  child implementation beads, not prose compaction.
- Do not let a high score hide uncertainty about graph shape. Synthesis should
  say whether the current bead is one reviewable PR-sized outcome or needs a
  split.
